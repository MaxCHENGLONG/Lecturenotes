Monte-Carlo vs Dynamic Programming

Dynamic Programming:

  • Bootstrapping: Each new estimate is based on a previous estimate.
  • Computes expectation exactly, but estimate since based on estimate V(St+1)V(S_{t+1})V(St+1​)﻿.
  • Needs model p(s′,r∣s,a)p(s',r|s,a)p(s′,r∣s,a)﻿ to compute expectation.

Monte Carlo:

  • We do not bootstrap, since we use the full return Gt.G_t.Gt​.﻿
  • Is an estimate because we use empirical mean of GtG_tGt​﻿, and not Eπ[Gt,St=s]\mathbb{E}_{\pi}[G_t,S_t=s]Eπ​[Gt​,St​=s]﻿.
  • No model needed, since samples GtG_tGt​﻿ can be computed from experience.

Monte Carlo (MC):

  • Only in episodic environments.
  • Only learn after every episode.
  • High variance, zero bias.
  • Converges to vπ(s)v_{\pi}(s)vπ​(s)﻿ (if α\alphaα﻿ decrease with a suitable rate)
  • Not very sensitive to initial conditions.
  • Usually more efficient in non-MDP environments.

Temporal Differences (TD):

  • Both episodic and continuing environments.
  • Can learn online after every step.
  • Low variance, but some bias.
  • Converges to vπ(s)v_{\pi}(s)vπ​(s)﻿ (if α\alphaα﻿ decrease with a suitable rate)
  • More sensitive to initial conditions.
  • Usually more efficient in MDP environments.

Batch MC and TD

  • MC and TD converge as experience →∞\rightarrow \infty→∞﻿.

  • Batch Learning: Repeatedly apply MC or TD to the episodes we have.
  • With constant (but small) α\alphaα﻿ both methods converge, but to different estimates.
