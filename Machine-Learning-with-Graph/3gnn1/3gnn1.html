Permutation Invariance (排列不变性)

Graph does not have a canonical order of the nodes!

We can have many different order plans.

What does it mean by “graph representation” is same for two order plans?

  • Consider we learn a function fff﻿ that maps a graph G=(A,X)G = (A,X)G=(A,X)﻿ to a vector Rd\mathbb{R}^dRd﻿ then
    • In other words, fff﻿ maps a graph to a d−dimd-dimd−dim﻿ embedding

  • Definition: For any graph function f:R∣V∣×m×R∣V∣×∣V∣→Rdf:\mathbb{R}^{|V| \times m} \times \mathbb{R}^{|V| \times |V|} \rightarrow \mathbb{R}^df:R∣V∣×m×R∣V∣×∣V∣→Rd﻿, fff﻿ is permutation-invariant if f(A,X)=f(PAPT,PX)f(A,X) = f(PAP^T,PX)f(A,X)=f(PAPT,PX)﻿ for any permutation PPP﻿.

Permutation Equivariance

  • For node representation: We learn a function fff﻿ that maps nodes of GGG﻿ to a matrix R∣V∣×d\mathbb{R}^{|V| \times d}R∣V∣×d﻿
    • In other words, each node in VVV﻿ is mapped to a d−dimd-dimd−dim﻿ embedding.
  • If the output vector of a node at the same position in the graph remains unchanged for any order plan, we say fff﻿ is permutation equivariant.
  • Definition: For any node function f:R∣V∣×m×R∣V∣×∣V∣→R∣V∣×df:\mathbb{R}^{|V| \times m} \times \mathbb{R}^{|V|\times|V|} \rightarrow \mathbb{R}^{|V|\times d}f:R∣V∣×m×R∣V∣×∣V∣→R∣V∣×d﻿, fff﻿ is permutation-equivariant if Pf(A,X)=f(PAPT,PX)P f(A,X) = f(PAP^T,PX)Pf(A,X)=f(PAPT,PX)﻿ for any permutation PPP﻿.
    • mmm﻿: each node has a m−dimm-dimm−dim﻿ feature vector associated with it
    • fff﻿ maps each node in VVV﻿ to a d−dimd-dimd−dim﻿ embedding.

Summary: Invariance and Equivariance

  • Permutation-invariant(Permute the input, the output stays the same)
  • Permutation-equivariant(Permute the input, output also permutes accordingly)
  • Examples:
    • f(A,X)=1TXf(A,X) = 1^TX f(A,X)=1TX﻿ Permutation-invariant
      • f(PAPT,PX)=1TPX=1TX=f(A,X)f(PAP^T,PX) = 1^TPX = 1^TX = f(A,X)f(PAPT,PX)=1TPX=1TX=f(A,X)﻿
    • f(A,X)=AXf(A,X) = AXf(A,X)=AX﻿ Permutation-equivariant
      • f(PAPT,PX)=PAPTPX=PAX=Pf(A,X)f(PAP^T,PX) = PAP^TPX = PAX = Pf(A,X)f(PAPT,PX)=PAPTPX=PAX=Pf(A,X)﻿

GCN: Invariance and Equivariance

Invariance and Equivariance Properties for a GCN?

  • Given a node, the GCN that computes its embedding is permutation invariant
  • Considering all nodes in a graph, GCN computation is permutation equivariant
